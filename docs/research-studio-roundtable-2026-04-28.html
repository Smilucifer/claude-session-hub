① 兆易创新走 3 轮（场景演示）

🟦 轮 1 · 默认 fan-out（三家独立隔离）
▶ 今天是否应该买入兆易创新？目前我有 5 万元仓位待入场。

↓ Hub 自动 fan-out，三家收到完全相同输入（含 profile.md 风偏 + 标准数据包），互不知情

Claude · 综合分析视角
"基本面：存储芯片周期下行尾部，2026 Q1 营收同比 +28% 但环比仍降。题材：DDR5 替代主线正宗。技术：放量突破 60 日均线但量比仅 1.4 偏温和。评分 65 / 仓位建议 1.5 万 / 入场分批"
Gemini · 实时情报视角（Google Search grounding）
"4/26 半导体协会公告 DDR5 国产化率达 18%，兆易创新被点名。但 4/27 美光 Q1 财报 guidance 偏谨慎，板块估值压制。近期催化 mixed"
Codex · 量化技术视角
"RPS 78（行业前 22%）/ MA20 上方 7 天 / 量比 1.4 偏温和 / RSI 62 未超买。回测同形态 60 日上涨概率 58%。技术分 70 / 启动趋势已立但未确认"

✅ 三家本色发挥：Claude 综合 / Gemini 联网 / Codex 量化 — 你能看到三个独立角度。

🟨 轮 2 · @debate（中转对方观点 + 用户补充）
@debate 我刚看到龙虎榜，4/27 一机构席位买入 8000 万。▶

↓ Hub orchestrator 自动给每家拼装 prompt：
    = 用户原话 + 龙虎榜信息 + 另两家上一轮观点

Claude（看到 Gemini + Codex 上轮观点 + 龙虎榜）
"Gemini 提的美光 guidance 利空我之前没看到。Codex 的"启动未确认"我同意，但机构 8000 万买入是关键确认信号。综合修正：评分 65 → 70 / 仓位建议 1.5 万 → 2 万 / 但分批入场，盘中破 60 日均线止损。"
Gemini（看到 Claude + Codex 上轮观点 + 龙虎榜）
"Claude 的 65 分偏乐观未充分计价美光利空。Codex 的"形态未确认"我赞同。机构买入 8000 万但未涨停，可能是左侧布局而非右侧追击，不能作为今日强买信号。我维持谨慎，评分 55 / 建议观察一日。"
Codex（看到 Claude + Gemini 上轮观点 + 龙虎榜）
"Claude 强调题材正宗合理。Gemini 提的美光利空压制估值上限。机构 8000 万 / 总成交额 = 量化资金占比 12%，显著但非压倒性。补充：5 日量能比已转正。技术分 70 → 73 / 但仓位建议不超 1.5 万。"

✅ 三家观点开始分化：Claude 转积极（70/2万）、Gemini 转谨慎（55/观望）、Codex 中性（73/1.5万）。这就是辩论价值。

🟩 轮 3 · @summary @claude（让 Claude 综合决策）
@summary @claude▶

↓ Hub 给 Claude 一份大 prompt：历史所有轮次三家观点 + profile.md + 强制输出最终决策

Claude · 最终决策报告
"结论：买入 1 万元（占 5 万仓位 20%）

三方共识：启动趋势已立、题材正宗、机构入场
三方分歧：Gemini 因美光利空更谨慎（55 分），Claude/Codex 给 70-73 分
降仓位理由：采纳 Gemini 的"美光 guidance 利空未充分计价"，将 Codex 上限 1.5 万再下调到 1 万，留 4 万作回踩加仓
止损：跌破 60 日均线（约 -4%）止损出场
加仓位置：回踩 5 日均线不破且缩量 → 加 1.5 万；放量突破前高 → 加 2 万"

✅ Claude 此时已有所有历史上下文，给出综合决策 + 加减仓路线图。

关键观察：每一轮都是用户驱动。Hub 不强制 debate 也不自动 summary。轮次可以无限延伸，也可以在轮 1 直接 @summary 草草结束。这就是 AI-Arena 范式的精髓。

② AI 角色定位（不强行分化，本色发挥）

三家收到完全相同的 system prompt。不扮演任何角色名。本色发挥让模型的训练分布做工。但 system prompt 里点明各自的天然优势，引导模型把强项亮出来：

Claude
天然强项：长链推理、综合判断、报告撰写
引导话术："你擅长把多源信息综合成一致框架，给出清晰的仓位/止损/加仓路线图"
常被 @summary：因为综合能力强
Gemini
天然强项：Google Search grounding、实时联网
引导话术："你能查到最新政策/行业景气/公告/舆情，请补充其他模型可能未知的信息"
独家价值：实时情报，避免训练截止盲区
Codex
天然强项：代码生成、量化逻辑、数据校验
引导话术："你可以写脚本计算 RPS/MA/量比/RSI，做形态识别和回测验证"
独家价值：把模糊判断量化为数字

所有三家共同读取 .arena/profile.md（你的风偏：基本面 15% + 预期差 35% + 资金面 30% + 技术面 20% + 红线 + 报告格式）。

④ 触发语法

语法语义示例

⑤ 数据接入策略（请你选其一）

三家辩论的共同事实底座从哪里来？

D1 推荐 Hub 自动调 LinDangAgent /s
轮 1 用户提问含股票名时，Hub 后台调 /s 拉 33 字段标准数据包，注入到三家的问题前缀。
✅ 三家看完全相同事实底座 ✅ 复用 LinDangAgent 五层兜底 ✅ 用户无感
⚠ 需要 Hub 能调用 LinDangAgent skill（一次性接入工作 ~0.5 天）
D2 三家各自独立拉数据
让每家自己用 akshare/tushare 等拉。
⚠ 三家事实底座可能不一致（数据源/缓存差异），辩论变成"数据吵架" ⚠ Codex 容易但 Claude/Gemini 没工具
D3 用户手动提供
用户在轮 1 把数据粘贴到提问里。
✅ 最简单 ⚠ 用户负担重，每次都要人工拉数据

观点抓取（关键技术点）

怎么从 PTY 输出中提取每家的"最终回答"？

  • Claude：复用现有 turn-complete 事件（claude --resume 输出 JSONL，Hub 已监听）
  • Gemini：CLI 输出有清晰的 prompt → output 边界，从 PTY scrollback 取末段（已在主驾审查里验证可行）
  • Codex：同 Gemini，从 transcript 取末段
  • 三家全部完成后才触发 next turn 状态转移；超时/失败有兜底（30s/60s）
