字段逐个处置

字段处置下游消费处置理由
personality 清空 Hub JS（_writePromptFile / _writeGeminiConfig / _writeCodexPersona）+ Python prompt_builder.py 每条 persona 指令都是对 native 能力的扭曲（例：小火龙"敢于提出不太靠谱的想法"鼓励幻觉；皮卡丘"500 字以内"硬截断深度分析）
system_prompt 清空 仅 Python prompt_builder.py 20-30 行长指令（工作方式/协作/输出要求），是人设强化层，一并清除
mes_example 清空 [] Python prompt_builder.py（[例句]） 示范发言本质是"模仿这种语气"，与放任 native 能力冲突
post_history_instructions 清空 Python prompt_builder.py（[提醒]） 压轴再强调一次人设，冗余
guardrails 清空 [] Python prompt_builder.py（[守则]） "不编造 / 不确定时说不确定"Claude/Gemini/Codex native 都做得比手写指令好；"不超过 400 字"等硬上限反而害事
id 保留 @mention 路由 key 功能必需
display_name 保留 prompt 头部"你是 X"+ UI 展示 保留宠物名身份识别；模型知道自己被叫"皮卡丘"但无 persona 指令 → 基本无扭曲
backing_cli / model 保留 后端路由（Claude / Gemini / Codex）+ 模型选择 功能必需
roles / strengths / caution / evaluation / tags 保留（元数据） 无代码消费（grep 确认） 纯 yaml 文档性字段，供未来 UI 角色卡或外部引用；不进 prompt 所以不影响模型

旧版 Claude prompt（~1800 字符）

# 皮卡丘 — AI Team Room

你是 皮卡丘，一个 AI 团队成员。

## 性格
务实、积极、乐观的实干派。团队里最能打硬仗的角色。
面对困难问题不回避，第一反应是"我来搞定"而不是"这很难"。
对自己的能力有充分自信，敢于承担艰巨任务。
说到做到，给出的方案一定是自己有把握落地的。
乐观但不盲目——乐观来自于扎实的能力，不是无知。

## 团队协作规则

- 你在房间 X 中与其他 AI 角色协作讨论。
- 收到队友或用户的消息后，认真思考并给出你的观点。
- 保持你的角色特征和说话风格一致。
