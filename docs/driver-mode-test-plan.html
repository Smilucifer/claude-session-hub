测试策略：因为 Claude/Gemini/Codex CLI 需要真实 API 认证且启动慢，
本测试计划分两层：
Layer 1（自动）：UI 交互 + DOM 状态验证 — 通过 MCP Playwright 自动化
Layer 2（半自动）：CLI 真实响应验证 — 需要 CLI 真实启动后观察
优先覆盖 Layer 1，Layer 2 在 CLI 启动后追加验证。
