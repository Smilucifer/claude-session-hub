Step 8: 预览文本提取优化方案

当前问题

左侧 session 列表的消息预览提取不准确：

  • 现状：从 pty 原始字节流中 strip ANSI → 取最后一行 > 3 字符
  • 结果：经常提取到提示符、状态栏、ANSI 残留（如 <u>1u>4;2m）
  • 期望：提取真正有意义的内容，如 你好！有什么可以帮你的？

调研关键发现：xterm.js Buffer API

xterm.js 内部有完整的 VT 状态机，terminal.buffer.active 存储的是最终渲染结果（纯文本），无需手动剥离 ANSI。相当于前端版的 tmux capture-pane。

const buffer = terminal.buffer.active;
const line = buffer.getLine(lineIndex);
const text = line.translateToString(true); // 纯文本，零 ANSI

为什么比 pty 流解析可靠：pty 流是碎片化字节流（一行可能分多个 chunk），而 xterm.js buffer 已完成所有渲染计算（光标移动、擦除、滚动等），读出来就是屏幕上看到的内容。

修复方案：前端 buffer 扫描 + 噪声过滤

之前（服务端 pty 流）

pty 字节流 → strip ANSI（不完整）→ 取最后一行

❌ ANSI 残留
❌ 碎片化
❌ 取到提示符

之后（前端 buffer 读取）

xterm.js buffer → 纯文本行 → 噪声过滤 → 有意义内容

✅ 零 ANSI
✅ 完整行
✅ 智能过滤

实现步骤

1. 新增前端函数：extractPreview()

文件：src/client/components/TerminalPanel.tsx

从 xterm.js buffer 底部（光标位置）向上扫描，跳过噪声行，返回第一行有意义内容。

噪声行过滤规则

噪声类型识别特征
空提示符/^\s*[>❯\$]\s*$/
PS 提示符/^\s*PS [A-Z]:\\/
user@host/\w+@\w+.*[~\/\\]/
Claude 状态栏/\[Opus|Sonnet|Context|Usage/
权限行/bypass permissions|allowed tools/i
配置信息/CLAUDE\.md|hooks|resets in/
分隔线/^\s*[━─═]{3,}/
空行/^\s*$/

2. 触发时机

当收到 session-updated 且 status 变为 idle 时，从对应 terminal 的 buffer 提取预览，通过 WebSocket 发回服务端更新。

3. 服务端配合

  • ClientMessage 新增：{ type: 'update-preview', sessionId, preview }
  • 服务端收到后更新 session.info.lastOutputPreview 并广播
  • 移除服务端的 stripAnsi + 行提取逻辑

4. 验证计划

  1. 新建 session → 启动 Claude Code → 发消息 → 预览显示 Claude 的回答
  2. 普通 PowerShell 命令（dir、echo hello）→ 预览显示命令输出
  3. 多 session 切换 → 预览不闪烁不乱跳
  4. Playwright 自动化测试全流程

总结

架构转变：预览提取从服务端（pty 流解析，脆弱）→ 前端（xterm.js buffer 读取，可靠）

核心优势：xterm.js 已经帮我们做了所有 ANSI 解析、光标计算、行合并，我们只需读取最终渲染结果并过滤噪声。
